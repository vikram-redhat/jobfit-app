# JobFit — Handover Document

## What it is

AI-powered job application tool for young/casual job seekers (under 30). Users paste a job description and get a fit score, tailored resume, and cover letter in ~20 seconds. Monetised via a freemium subscription model.

**Production URL:** https://jobfit.today
**Paid-traffic landing page:** https://jobfit.today/start (TikTok bio, Meta ads, Google Ads — robots-noindex'd)
**Repo:** GitHub (deployed via Vercel — auto-deploys on push to `main`)
**Admin email:** admin@craftgossip.com  *(controls /admin access; separate from user-facing help@jobfit.today)*
**User-facing email:** help@jobfit.today  *(contact form recipient + privacy policy contact)*

## Content (programmatic SEO)

The repo contains a content cache that drives the 200+ resume guide pages at `/resume-for/<slug>`.

**Files:**
- `content/roles.json` — curated dataset of 218 role/situation entries across 9 categories. Append new entries to the end (don't reorder — slugs are permanent URLs).
- `content/roles/<slug>.json` — per-page content cache, generated by Claude Haiku at `npm run content:generate` time. **These files MUST be committed to git** — Vercel can't generate them at build time, only read them.

**Categories** (in display order on the `/resume-for` hub):
Life situations · Tech · Healthcare · New grad · Business · Creative · Service · Trades · Education

**Generator script:** `scripts/generate-role-content.js` — see "Operational Commands" below.

**Cost:** ~$1–3 in Anthropic API spend to generate all 218 roles (~3000 output tokens × Haiku pricing × 218). One-time, idempotent on re-run.

## User Flow

```
Landing page (/) → Sign up / Google OAuth
  → Email confirmation (if email/password)
  → /onboarding — upload PDF resume or fill 4-step form
  → /dashboard — paste job description → AI analysis → fit score saved
  → /job/[id] — view score, strengths, gaps, angle, key requirements
              → generate tailored resume → edit → download PDF
              → generate cover letter → edit → download PDF
  → Track application status (New → Applied → Interview → Offered/Rejected/Dismissed)

Free limit hit → /upgrade → Stripe checkout → /dashboard?upgraded=1
```

## Stripe Integration

- **Checkout:** `/api/stripe/checkout` creates a session with promo code support. Creates a Stripe customer if one doesn't exist, stores `stripe_customer_id` on the profile.
- **Webhook:** `/api/stripe/webhook` handles:
  - `checkout.session.completed` → sets `is_subscribed: true`
  - `customer.subscription.updated` → syncs subscription status
  - `customer.subscription.deleted` → sets `is_subscribed: false`
- **Billing portal:** `/api/stripe/portal` — lets Pro users manage/cancel subscription
- **Promo codes:** Created and managed from the Admin panel, applied at Stripe checkout

Webhook secret must be set in both Vercel env vars and the Stripe dashboard (for the production endpoint `https://jobfit.today/api/stripe/webhook`).

## Paywall Logic

- `profiles.analysis_count` increments on each analysis and never decrements (even if jobs are deleted)
- `app_settings` table holds `free_tier_limit` (default: 2)
- Users who previously deleted their account and re-signed up are detected via `deleted_accounts` table and get 0 free analyses
- Paywall returns HTTP 402 → client redirects to `/upgrade`

## SEO / GEO

Full SEO and Generative Engine Optimization setup:

- **Metadata:** `layout.js` exports a full Next.js `metadata` object — title template, description, OG tags, Twitter card, robots directives, `og.png` social image (static `public/og.png`, the "product card" design with mock fit-score)
- **robots.txt** (`robots.js`): blocks auth-gated routes + `/start`, explicitly allows 14 AI crawlers by name (GPTBot, ClaudeBot, PerplexityBot, Google-Extended, Applebot-Extended, etc.)
- **sitemap.xml** (`sitemap.js`): pulls all buildable role guides dynamically. Sitemap entry count grows as more roles are generated and committed.
- **llms.txt / llms-full.txt**: the emerging standard for AI search engines. Short version is a TL;DR; full version includes competitor comparison, FAQs, use cases, and references the resume-guides + free-tools surfaces — all written for accurate citation by ChatGPT/Perplexity/Claude/Gemini
- **JSON-LD** (`JsonLd.js`): `Organization`, `SoftwareApplication` (with Free + Pro `Offer` nodes), and `FAQPage` structured data on the landing page. Tool pages and role pages each emit their own `FAQPage` + `Article` + `BreadcrumbList` blocks
- **Free tools** (`/tools`): two anonymous, no-signup tools (keyword extractor, resume grader) that generate shareable result permalinks at `/tools/[tool]/r/[id]` — server-rendered, individually indexed. Each result page has unique `<title>/<description>` generated from the actual AI output. Acts as top-of-funnel organic/AI search traffic with CTAs back to the main app
- **Anonymous tool endpoints** (`/api/tools/*`): rate-limited at 10 req/IP/hour, prompt-injection sanitized, degrade gracefully if DB insert fails
- **Programmatic resume guides** (`/resume-for/<slug>`): 218 role/situation guides pre-rendered at build time via `generateStaticParams`. Each page contains: top-skills list, 3 weak/strong bullet rewrites, common mistakes, structure tips, ATS keywords, salary signal, FAQ, related-role cross-links. Two prompt flavors in the generator: ROLE_SYSTEM_PROMPT for job titles, SITUATION_SYSTEM_PROMPT for life-situation guides (career change, after layoff, returning to work, etc.). Page template branches on `content.category === 'situation'` to swap headings appropriately.
- **Internal linking:** every public page reaches every other public page in ≤ 2 clicks. Homepage footer + nav, SiteHeader on all sub-pages, `/resume-for` hub lists every cached role grouped by category, role pages link to 6 sibling roles + breadcrumb back to hub, tool pages link contextually to `/resume-for`. Logged-in surfaces also link out: `Nav.js` has "Resume guides" (md+ visible), `/dashboard` empty state has secondary link.

### Search Console / Bing submissions

- Google Search Console: sitemap submitted at `https://jobfit.today/sitemap.xml`. The list view sometimes shows "Couldn't fetch" while the detail view shows "Sitemap processed successfully" with the correct discovered-page count — this is a known GSC quirk and not a real failure. Trust the discovered-pages number, not the red status.
- Bing Webmaster Tools: submit the same sitemap. Bing powers ChatGPT search, so this is high-leverage.
- IndexNow: enable from Bing Webmaster for fast-indexing across Bing, Yandex, Naver.

### What's NOT submitted to search engines

- `/start` is `robots: noindex` and explicitly disallowed in robots.txt — paid LP only, must not compete with `/` for ranking.
- Tool result permalinks (`/tools/*/r/[id]`) are NOT enumerated in the sitemap (ephemeral, 30-day TTL via Supabase) but remain crawlable through internal links once shared.

## Two landing pages

Critical to understand: the site has TWO landing surfaces, optimized for different traffic sources.

### `/` — organic homepage
- For Google / Bing / AI search traffic
- Mobile order (top to bottom): nav → eyebrow → headline → subhead → free-tools off-ramp link → above-the-fold "Get started free →" button (smooth-scrolls to form) → 3 feature blocks → demo GIF → pricing line → auth form (#auth-form anchor) → sticky bottom CTA bar (Get started + Free tools)
- Desktop: traditional left-right split, value prop on left, auth form on right rail (lg:w-[420px])
- Discoverability of free tools and resume guides is high (header nav + footer + inline links + sticky bar) — for cold visitors who aren't ready to sign up

### `/start` — paid-traffic landing page
- For TikTok / Meta / Google Ads traffic — visitors are pre-warmed by the ad creative
- Mobile + desktop both: minimal logo-only header → centered headline → centered subhead → AUTH FORM FIRST → 3 short proof points underneath → footer
- Robots-noindex'd + canonical to `/`. Excluded from sitemap. Disallowed in robots.txt. *Will not* compete with `/` for organic ranking.
- POINT ALL PAID ADS TO `/start`, not `/`. The conversion delta is meaningful (15-30%+ for warm traffic).

Both pages render `<AuthForm />` from `src/components/AuthForm.js` so the auth flow can never drift between them. Change auth UX in one place, applies everywhere.

## Public-page header consistency

`src/components/SiteHeader.js` is the single source of truth for the public header. Used by `/tools/*`, `/resume-for/*`, `/contact`, `/privacy`. Renders: `JobFit.today` wordmark + 3-item nav (Resume guides · Free tools · Sign in).

The homepage `/` and the paid LP `/start` have **custom headers** because their layouts are structurally different. The homepage has 2-3 items (no "Sign in" because the page IS the sign-in form). `/start` has logo only (paid LPs intentionally remove escape hatches).

If you ever need to add a new nav item, change `SiteHeader.js` once and it propagates. Don't duplicate header markup inside individual page components — that's how nav drift happens (we hit this multiple times).

The "Contact" link is **deliberately absent from the header** — it's discoverable via every footer instead. Header stays at 3 items max.

## Operational commands

### Generating role-page content (run locally; commit & push)

```bash
# Show how many of the 218 roles have cached content
npm run content:status

# Generate content for any roles missing a cache file (idempotent)
npm run content:generate

# Smoke-test on N roles
node scripts/generate-role-content.js --limit=5

# Generate one specific role only
node scripts/generate-role-content.js --only=barista

# Force-regenerate one role (deletes the cache, re-runs Claude)
node scripts/generate-role-content.js --regenerate=software-engineer
```

**IMPORTANT:** Vercel CANNOT generate role content during build — its filesystem is read-only after build, and resets between deploys. The content generation must run locally, the resulting JSON files must be committed to git, and Vercel reads them during `next build` to pre-render the static pages.

```bash
# After generating, commit + push to trigger a Vercel build
git add content/roles.json content/roles/
git commit -m "Generate role-page content"
git push
```

### Adding a new role

1. Append an entry to `content/roles.json` (don't reorder existing entries — slugs are URLs).
2. `node scripts/generate-role-content.js --only=<your-new-slug>`
3. Verify the resulting `content/roles/<your-new-slug>.json` looks good.
4. Commit and push.

### Tweaking the prompt

Edit `SYSTEM_PROMPT` (or `SITUATION_SYSTEM_PROMPT`) in `scripts/generate-role-content.js`. To regenerate everything with the new prompt, delete `content/roles/*.json` and re-run `npm run content:generate`. Costs another ~$1-3.

### Deprecating a role

Set `"deprecated": true` on the entry in `content/roles.json`. Don't delete the entry — that frees up the slug to be reused, and the URL is permanent. Deprecated entries are filtered out of `getBuildableRoles()`.

## Known Notes

- PDF download is implemented as a print-ready HTML file (not a true server-side PDF). The user opens it in the browser and uses Ctrl+P / Print to save as PDF.
- Google OAuth requires the callback URL `https://jobfit.today/auth/callback` to be configured in both Supabase (Authentication > URL Configuration) and Google Cloud Console.
- The demo GIF (`/public/demo.gif`) is shown on mobile only (hidden on desktop via Tailwind `lg:hidden`).
- `ToolForm` is a Client Component that imports result components directly — do not pass `renderResult` as a function prop from Server Component pages (Next.js App Router does not allow function props across the server/client boundary).
- The `/start` paid LP is robots-noindex'd AND in the robots.txt disallow list. Both are deliberate. Don't link to it from anywhere on the site (it's only reached via paid ad URLs).
- `roles.json` slug ordering is irrelevant for display (categories control rendering order on the hub) but is permanent for URL stability. Append new roles to the end.
- `content/roles/*.json` files MUST be committed to git. They're not generated during Vercel build.
- Two prompt flavors live in `scripts/generate-role-content.js`: `ROLE_SYSTEM_PROMPT` and `SITUATION_SYSTEM_PROMPT`. They produce the same JSON schema, but with different framing (job-title vs life-situation). The `/resume-for/[role]` page template branches on `content.category === 'situation'` to swap section headings ("Top skills hiring managers look for" → "What to lean on", "Common mistakes on a X resume" → "Common mistakes to avoid", etc.).
- The Supabase admin user (admin@craftgossip.com) is a separate concept from the user-facing help@jobfit.today email. The admin email controls `/admin` access via the `ADMIN_EMAIL` env var. The help email is the contact-form recipient and the privacy-policy contact.
- Anonymous AI calls cost real money. The rate limit is 10/IP/hour by default in `src/lib/rate-limit.js`. If you ever see API spend climbing on the tools, check this first.
